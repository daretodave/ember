# Phase 4 ŌĆö Magic-link auth

> Brief generated by the autonomous loop.

## Outcome

`/signin` page is live; Supabase Auth magic-link round-trip works;
protected routes (`/today`, `/log`, `/settings`) redirect unauthenticated
users to `/signin`; authenticated users can sign out.

## Why

Phase 3 shipped the anonymous landing with a "sign in to start" CTA.
The CTA links to `/signin` which doesn't exist yet. Phase 4 closes that gap
and makes auth the foundation for Phases 5ŌĆō8 (all of which require a session).

## Scope (must ship)

1. **`/signin` page** ŌĆö email form matching `design/Ember ┬Ę Signin.html`:
   - Single email input, "send the link" button
   - On submit: call Supabase `signInWithOtp`, render a "check your email"
     confirmation state in the same card (no full redirect)
   - On error: show the error message in a muted error card below the form
   - Already-authenticated users landing on `/signin` are immediately
     redirected to `/today` (handled in middleware)

2. **`/auth/callback` route** ŌĆö Next.js route handler that:
   - Receives the `?code=` query param from the magic-link email
   - Exchanges it for a session via `supabase.auth.exchangeCodeForSession(code)`
   - Redirects to `/today` on success, `/signin?error=auth` on failure

3. **`/auth/signout` route** ŌĆö POST-only route handler:
   - Calls `supabase.auth.signOut()`
   - Redirects to `/`

4. **`src/middleware.ts`** ŌĆö Next.js middleware:
   - Protected paths: `/today`, `/log`, `/settings` (and sub-paths)
   - Public paths: `/`, `/signin`, `/auth`, `/u`
   - For protected paths: if no session, redirect to `/signin`
   - For `/signin`: if session exists, redirect to `/today`
   - Refreshes stale access tokens on every request (Supabase SSR pattern)

5. **Tests**:
   - Unit: `src/app/signin/__tests__/actions.test.ts` ŌĆö mocks Supabase,
     verifies `signInWithOtp` is called with the right email
   - E2E: `apps/e2e/tests/signin.spec.ts` ŌĆö renders `/signin`, verifies
     heading, verifies form elements, verifies 200 status.
     Also verifies `/today` redirects to `/signin` when no session.

## Decisions (standing)

- No password flow. Magic-link only, per `bearings.md` "Auth posture".
- Redirect URL for the magic link callback: `<NEXT_PUBLIC_SITE_URL>/auth/callback`.
  Falls back to `http://localhost:3000/auth/callback` for local dev.
- The email sent by Supabase Auth uses Supabase's default template ŌĆö no
  custom template in v1 (per `design/CLAUDE.md` "What the design folder does NOT cover").
- Rate limit on magic-link send is enforced by Supabase (3 per email per 24h
  per `bearings.md`). No additional client-side throttle in v1.
- Sign-out is a form POST (no client-side JS required) ŌĆö simple `<form
  action="/auth/signout" method="post">` on authenticated pages.
- The `/auth/signout` handler accepts POST only; GET returns 405.
